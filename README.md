# JamBuddy 🎸🥁 — Telegram Bot & Mini App Backend

**JamBuddy** — это полноценная платформа внутри Telegram, которая помогает музыкантам находить друг друга поблизости, объединяться в группы, выбирать инструменты и отправлять приглашения на джемы в реальном времени.

Проект представляет собой Spring Boot приложение, которое одновременно управляет логикой Telegram-бота (через Long Polling/Webhooks) и отдает Web-страницы (HTML/JS) вместе с REST API для **Telegram Mini App**.

---

## 🗺 Как это устроено (Архитектура)

Приложение связывает три слоя:
1. **Telegram Bot:** Точка входа. Приветствует пользователя, обрабатывает команды и предлагает открыть встроенное приложение (Mini App).
2. **Telegram Mini App (Frontend):** Встроенный в Telegram веб-интерфейс (HTML/JS), где музыканты видят интерактивную карту/список, настраивают профили и управляют инвайтами.
3. **Spring Boot Бекенд:** * Обслуживает статику (фронтенд Mini App).
   * Предоставляет REST API для JS-скриптов приложения.
   * Безопасность: Авторизация базируется на валидации `initData` от Telegram WebApp API.
   * Уведомления: Использует **Server-Sent Events (SSE)** для мгновенной доставки инвайтов на джемы прямо внутри Mini App без перезагрузки страниц.

---

## 🚀 Функционал платформы

* **Управление профилем:** Создание анкеты музыканта (имя, возраст, описание, telegram-аккаунт) с указанием точной геопозиции (координаты) и выбором музыкальных инструментов (гитара, барабаны и т.д.).
* **Умный поиск:** Поиск музыкантов в определенном радиусе с фильтрацией по инструментам и возрасту.
* **Система инвайтов (приглашений):** Возможность отправить музыканту инвайт на совместный джем, отменить его или изменить статус (Принять/Отклонить).
* **Мгновенные SSE-уведомления:** Если вам прислали инвайт, приложение сразу же отобразит его на экране в реальном времени.

---

## 🛠 Технологический стек

* **Backend:** Java 17, Spring Boot 3.2+, Spring Data JPA, Spring Security (JWT / Telegram WebApp Auth Validation).
* **Frontend:** HTML5, CSS3, JavaScript (Telegram WebApps API, SSE Client).
* **База данных:** PostgreSQL / H2.
* **Тестирование:** JUnit 5, Mockito, MockMvc (полное изолированное покрытие контроллеров и бизнес-логики).
